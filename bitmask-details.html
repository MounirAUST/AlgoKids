Bitmask

1. What is a Bitmask?

A bitmask is a technique that uses bits (0s and 1s) to represent and manipulate sets or states. It allows us to perform set operations efficiently using binary numbers.

2. Why Bitmask?

Bitmasking is useful when you want to:

  • Represent subsets (especially in DP or combinatorics)
  • Toggle, set, or clear specific states
  • Optimize performance by using fewer variables

3. Common Operations

  • mask | (1 << i) → set the i-th bit to 1
  • mask & ~(1 << i) → clear the i-th bit to 0
  • mask ^ (1 << i) → toggle the i-th bit
  • mask & (1 << i) → check if the i-th bit is set

These use bitwise operators: | OR, & AND, ^ XOR, ~ NOT.

4. Visual Exploration

Try changing bits interactively in the visualizer. You can:

  • Set, clear, or toggle bits
  • Observe the binary representation
  • See how subsets change based on binary

5. Summary

Bitmasking is a smart way to pack multiple boolean states into a single number. It’s fast, memory-efficient, and essential in many algorithmic problems like subset generation and dynamic programming.
