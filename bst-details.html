Binary Search Tree & AVL Tree

1. What is a Binary Search Tree (BST)?

A Binary Search Tree is a tree structure where each node has at most two children. For every node:

  • All values in the left subtree are less than the node.
  • All values in the right subtree are greater than the node.

This property allows for efficient search, insert, and delete operations.

2. Why BST is Important

BSTs make it easier to organize and retrieve data quickly. In the best case, they allow:

  • Searching in O(log N)
  • Inserting in O(log N)
  • Removing in O(log N)

However, if the tree becomes unbalanced, performance can degrade to O(N).

3. Introducing AVL Trees

An AVL Tree is a self-balancing Binary Search Tree. It automatically keeps the height difference (balance factor) between the left and right subtree of any node to no more than 1.

This guarantees that operations remain efficient and balanced.

3-1. AVL Rotations

  • Right Rotation (LL Case)
  • Left Rotation (RR Case)
  • Left-Right Rotation (LR Case)
  • Right-Left Rotation (RL Case)

These rotations are performed to rebalance the tree whenever needed after insertions or deletions.

3-2. Why AVL Trees Matter

AVL Trees prevent the BST from becoming skewed. This keeps operations consistently fast.
